# TASK: Outbox 4xx dead-letter

> Stop retrying outbox pushes the host rejected with a 4xx; only 5xx/timeouts retry.

## Context

botcity is adding hard validation on `/api/rooms/close` (DESIGN-PEOPLE.md §4.3 in the
botcity repo): a malformed board gets a deterministic 400. Today `flushOutbox` retries
ANY non-2xx forever with capped backoff, and close payloads are regenerated byte-identical
from write-once `coingame_board` — so a 400 would become an infinite hourly hammer.
This change MUST deploy before botcity's validation goes strict (rollout order is a
recorded decision).

## Requirements

1. In `lib/outbox.ts` `flushOutbox`, a response with status 400–499 marks the row
   dead-lettered: it is never retried again.
2. Dead-lettering is visible: add a `dead_at timestamptz` column to `coingame_outbox`
   and log one loud `console.error` including kind, room_id, status, and response body
   (first ~500 chars).
3. 5xx, network errors, and timeouts keep the existing backoff behavior unchanged.
4. The due-rows query excludes dead rows (`dead_at is null`).
5. Schema change ships in both places: `db/schema.sql` AND an idempotent statement in
   `db/migrate-additive.mjs` (the additive script's `coingame_` prefix self-guard must
   still pass).

## Implementation Notes

- File: `lib/outbox.ts` — the try/fetch block already has `res.ok`; capture `res.status`
  and branch: `ok` → delivered, `status >= 400 && < 500` → dead, else → backoff.
- Read the response body with `await res.text().catch(() => "")` only on the 4xx branch.
- Migration: `alter table coingame_outbox add column if not exists dead_at timestamptz`.
- No new dependencies, no new files.

## Do Not Change

- `lib/adjudicate.ts` — board computation and tie-break are correct and load-bearing.
- The signing scheme (raw-body HMAC, `x-rooms-signature`) and frozen wire names
  (`roomId`, `t`, `ROOMS_SIGNING_KEY`, `X-Rooms-Timestamp`, `X-Rooms-Signature`,
  `POST /api/rooms/close`, `d-YYYY-MM-DD` refs).
- `db/schema.sql` beyond the one column; never touch non-`coingame_` objects.
- Backoff curve in `backoffSeconds` — unchanged for retryable failures.

## Acceptance Criteria

- [ ] `npm run build` passes with zero errors.
- [ ] A mocked 400 response marks the row `dead_at` set, logs the error, and the row is
      skipped by the next `flushOutbox`.
- [ ] A mocked 503 response still reschedules with backoff exactly as before.
- [ ] `node db/migrate-additive.mjs` runs clean and is idempotent (run twice).
- [ ] `git diff` touches only `lib/outbox.ts`, `db/schema.sql`, `db/migrate-additive.mjs`.
